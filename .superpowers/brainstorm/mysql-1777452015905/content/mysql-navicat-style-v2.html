独立查询标签页的收益

数据页能留出更多横向空间给表格，查询页则可以放更大的编辑器和结果区域。这个拆法比把 SQL 压在底部更接近 Navicat 的使用习惯。

inspection 保存历史 结果多标签
SELECT id, device_code, status, updated_at FROM inspection.device_status WHERE city_name = '福州' ORDER BY updated_at DESC LIMIT 200; UPDATE inspection.device_status SET remark = '人工确认离线' WHERE id = 1024;
Result 1 返回 200 行。这里后续会放成真正的结果标签或表格区域，而不是压缩在数据页底部。
Result 2 影响 1 行。失败时按语句编号标记，并在这一页停住，不挤占表数据视图。
History 执行批次、操作者、时间和结果摘要都在后端持久化，后续可从左侧 Queries / History 打开。

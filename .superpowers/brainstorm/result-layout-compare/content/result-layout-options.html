查询结果区怎么拿到更多空间？

先选一个总体方向。重点看结果表格能不能更完整展示，而不是配色细节。

A. 折叠 SQL 编辑器
query_3.sql 收起编辑器
SELECT * FROM ...
结果表格

保留同页编辑

默认展开 SQL，执行后可以一键收起，上下文还在同一页里。

B. 结果独立标签页
query_3.sql
结果 #15
完整结果可横向滚动

查询和结果分离

SQL 编辑留在原标签，执行后新开结果标签页，结果可以占满主体区域。

C. 结果专注模式
query_3.sql
专注结果
结果全屏区

同标签切换视图

不新开标签，点击后把 SQL 编辑器几乎全部收起，只看结果，再切回编辑。

我建议先做 B

  • 结果区空间最大，最接近数据库客户端的使用习惯
  • 不会丢掉原 SQL 上下文
  • 后续还能叠加 C 的“专注模式”

你需要权衡

  • A 最省开发量，但结果空间提升有限
  • B 标签会变多，需要设计命名规则
  • C 交互最顺，但状态切换逻辑会更细
